Electricity Exchange Frontend Technical Challenge

Welcome

This is the technical challenge solution for Michael Jonker. I hope you enjoy evaluating it as much as I enjoyed building it.

Visual Design Methodology

I have approached the challenge as a Single Page Application (SPA), leaning heavily on the design patterns of Google Material Design.

These patterns have been implemented without the use of auto-generated templates or frameworks, ie. I have used bespoke HTML, JS and CSS.

Colors are based on the visual identity of the existing Electricity Exchange website.

Libraries and logic used

The primary app layout is pure HTML, which sets out the app layout areas.

Almost everything else are reusable components, the HTML for which are generated within the javascript using es6 template string syntax.

Every item created, eg. widget, button, menu item, input, is a prototypical object, with functions to control display, rendering, validation etc.

jQuery

I have used the jQuery library extensively for dom manipulation. Animations are CSS3

FontAwesome

I have used FontAwesome library for graphical icons

Functional Logic

All prototypical app elements are stored in a hierarchical tree. I have called this the "Virtual Dom" as it does not necessarily reflect the visual DOM, but contains references to all the visual DOM elements as they relate to the data structure they represent.

With this I am able to recursively query, validate and update the entire data-set and trigger any required visual updates along the way.

Notes on system

The application can be statically served, so it is OS agnostic. My personal preference is Debian based Linux systems.

For a real world production scenario, I would use a packaging framework to minify and package the application, then serve it from a fast static server such as nGinx.

Assumptions

The application was built for modern browsers using HTML5, CSS3 and ES6 features.

The structure is present for CSS @media query style mobile optimisation, but I have not undertaken this as part of the scope for this exercise.
